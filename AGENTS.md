# Agent rules for this repo

0. **The thoughts are mine, not yours.** This is MY commonplace book. Never write commentary, opinions, interpretations, or "half thoughts" on my behalf. Your job is: formatting and page structure, citation form, finding and verifying references, and conversation that enriches my knowledge so I can form my own view. If an entry needs commentary and I haven't supplied it, ask me for it (chat with me, take dictation, tighten my phrasing) rather than inventing it. Editing my prose for clarity is fine; originating the ideas in it is not.
1. **No em dashes. None.** Not `&mdash;`, not `—`. Restructure the sentence instead: use a period, a comma, a colon, or parentheses.
2. **Side-note placement.** Default to anchoring a margin note (`<label>/<input>/<span class="marginnote">`) at the top of the element it intellectually branches from. Most often that is the pull quote the entry is discussing, so the note's toggle belongs inside the blockquote's first paragraph, not in the commentary below it.
3. **Citations are numbered sidenotes, not source-block entries.** Scholarly corroboration (dictionary entries, glossaries, papers) goes in a numbered Tufte sidenote (`<label class="margin-toggle sidenote-number">/<input>/<span class="sidenote">`) anchored at the specific claim it supports. Never anchor a citation sidenote inside quoted text unless it really was that quote's source.
4. **Source block form.** The `<dl class="source">` at the bottom holds only provenance: a `Source:` row with a formal citation of the work the entry is about, then a lowercase `via:` row tracing how I personally found it (chained with commas, e.g. "Hacker News thread (date), via my own wiki-hn project"). The `via:` row is never the original publication, a prior edition, or the article's own source. Put those in `Source:` if needed. `via:` is the discovery path: the bookshop, link, conversation, search, newsletter, project, or memory that led me to the work. See `posts/nature-of-the-firm.html` for the canonical example.
5. **PDF quote verification.** When transcribing quotes from a PDF, verify wording against the original PDF page render. Do not treat OCR output as authoritative when validating quoted text.
6. **Illegible marginalia workflow.** If handwritten annotations are unclear, ask me to confirm the exact wording. Do not guess.
7. **Drafting workflow in this repo.** When I ask for a draft entry, draft directly in a new page under `posts/` and run the local dev server so I can review it in-browser.
8. **Preserve unrelated in-flight edits.** If another thread has already modified tags or taxonomy, keep those changes intact unless I explicitly ask to remove them.
